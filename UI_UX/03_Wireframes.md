# Wireframes – HimBhasha AI

## Overview

Low-fidelity ASCII wireframes for every page in HimBhasha AI. These wireframes define layout, content placement, and interaction zones — not visual styling. Refer to `04_Design_System.md` for typography, color, and spacing specifications.

All wireframes follow a **mobile-first** layout (375px width). Desktop layouts expand the content area horizontally while maintaining the same vertical structure.

# Wireframe Notes

| Guideline | Detail |
| --------- | ------ |
| Touch targets | Minimum 48×48px for all interactive elements |
| Content width | Max 640px centered on desktop |
| Card grid | 2 columns on mobile, 3–4 on desktop |
| Input bars | Fixed at bottom on mobile chat/voice screens |
| Back button | Top-left on all module pages |
| Empty states | Every module has a helpful empty-state message |
